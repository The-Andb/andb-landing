MacOS "App is damaged" fix

Seeing a "damaged" error? This is a common macOS Gatekeeper protection for non-notarized apps. To fix it, open Terminal.app and run:

xattr -cr /Applications/The\ Andb.app

This removes the quarantine attribute and allows the app to launch normally.

The Vision

"Every deployment feels risky.
I wanted to change that."

After years as a backend developer, I kept hitting the same wall: databases are hard to version. Manually comparing schemas, missing columns, changed data types, silent production breaks... the risks only grow as your project evolves.

I realized that while code has Git and CI/CD, the database side was still manual and fragile. I asked myself: "Can I build a version control tool for DDL, like Git does for code?"

What started as a small CLI tool to solve my own pains has evolved into The Andb. It’s a developer-first tool built to restore trust and visibility in your migrations.
