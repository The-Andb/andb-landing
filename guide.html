Getting Started Guide

Follow these simple steps to set up your environment and start migrating databases like a pro.

1

Bypass macOS Gatekeeper

Fix "App is damaged" Error

Since The Andb is built by an independent developer, macOS Gatekeeper might block it. This is normal. To fix the "damaged" warning, follow these steps:

  • 1

    Open Terminal.app (found in Applications > Utilities).

  • 2

    Copy and paste the command below, then press Enter:

Command:

xattr -cr /Applications/The\ Andb.app
Terminal Screenshot
2

Configure Connections

Screenshot Placeholder

Open the Connections settings to add your source and target databases. We currently support MySQL/MariaDB with PostgreSQL support coming soon.

  • Fill in Host, Port, User, and Password.
  • Test individual connections before proceeding.
3

Create Migration Pairs

Define which databases should be compared. You can create pairs like Dev → UAT or UAT → Prod.

The app uses these pairs to automatically manage schemas and generate migration scripts.

4

Compare & Sync

Navigate to the Compare tab, select your pair, and watch the magic happen.

You can review every change, see specific SQL differences, and migrate individual functions or entire schemas with a single click.
